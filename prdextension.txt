# ?? Product Requirements Document (PRD)
## Unified Order, Payment & Communication System (Shopify)

---

## ?? Objective

Create a consistent, scalable system across online and offline sales that:

- Handles stock + MTO (made-to-order) journeys
- Supports partial + full payments
- Ensures accurate invoicing (proforma vs final)
- Eliminates duplicate or incorrect customer communication
- Simplifies execution for store staff

---

## ?? Core Problem

- Orders behave inconsistently across online, offline, and payment links
- Draft vs Order confusion breaks workflows
- MTO pricing and attributes change post-order
- Partial payments are not structured
- Staff cannot handle JSON-based inputs
- Communication triggers are event-based, not context-aware

---

## ?? Solution Overview

Shift system from **order-state driven ? data-driven**

### 3 Core Layers:

#### 1. Payment Ledger
- amount_paid  
- amount_pending  
- payment_status (partial / full)

#### 2. Product Finalization
- jewel_code  
- gross/net weight  
- diamond & gemstone details  
- is_finalized (true/false)

#### 3. Communication Engine

Uses 3 Shopify constructs:

- ?? Draft Order Invoice ? deposit / partial payment  
- ? Order Confirmation ? proforma OR final invoice  
- ?? Shipping Confirmation ? only when shipped

## ?? Payment Link System

Using GoKwik API:

- Endpoint: `/v1/payments/links`
- merchant_reference_id = Shopify Order ID

### Flow:
1. Generate link using amount_pending
2. Send to customer
3. On payment success (webhook/polling):
   - Update amount_paid
   - Update amount_pending
   - Update payment_status
   - Trigger correct email
